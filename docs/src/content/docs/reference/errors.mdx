TensorDB uses the `TensorError` enum for all error conditions.

## Error Types

```rust
pub enum TensorError {
    /// I/O errors (disk, WAL, SSTable)
    Io(std::io::Error),

    /// Configuration validation failure
    Config(String),

    /// SQL parsing or execution error
    Sql(String),

    /// Key not found
    NotFound(String),

    /// Serialization/deserialization error
    Serde(String),

    /// Internal engine error
    Internal(String),
}
```

## Handling Errors

```rust
use tensordb::{Database, Config, TensorError};

match db.sql("SELECT * FROM nonexistent", &[]) {
    Ok(results) => println!("{:?}", results),
    Err(TensorError::Sql(msg)) => eprintln!("SQL error: {}", msg),
    Err(TensorError::Io(e)) => eprintln!("I/O error: {}", e),
    Err(e) => eprintln!("Other error: {}", e),
}
```

## Common Errors

| Error | Cause | Solution |
|-------|-------|---------|
| `Config("memtable_max_bytes must be >= 1024")` | Invalid config | Fix the configuration value |
| `Sql("table 'x' not found")` | Missing table | CREATE TABLE first |
| `Sql("parse error")` | SQL syntax error | Check SQL syntax |
| `Io("Permission denied")` | File permissions | Check data directory permissions |
| `Io("No space left on device")` | Disk full | Free disk space |

## Result Type

```rust
pub type Result<T> = std::result::Result<T, TensorError>;
```

All TensorDB functions return `tensordb::Result<T>`, which is an alias for `std::result::Result<T, TensorError>`.
